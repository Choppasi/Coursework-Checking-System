# Техническое задание: Система управления курсовыми работами

## 1. Общие сведения

### 1.1. Назначение системы
Веб-приложение для управления курсовыми работами студентов с поддержкой групповой структуры, планирования задач по пунктам курсовой работы, контроля дедлайнов и прикрепления результатов.

### 1.2. Целевая аудитория
- **Студенты** — выполняют пункты курсовой работы, прикрепляют результаты, отслеживают прогресс
- **Преподаватели/Руководители** — создают планы курсовых, назначают дедлайны, проверяют результаты
- **Администраторы** — управляют группами и пользователями

### 1.3. Технологический стек
- **Backend**: Go (Golang)
- **База данных**: PostgreSQL
- **Frontend**: HTML, CSS, JavaScript (Vanilla)
- **HTTP Router**: Gorilla Mux
- **Миграции**: SQL-скрипты

## 2. Функциональные требования

### 2.1. Управление пользователями

#### 2.1.1. Ролевая модель
| Роль | Права |
|------|-------|
| Admin | Управление группами, пользователями, всеми курсовыми |
| Teacher | Создание планов курсовых, назначение студентов, проверка работ |
| Student | Выполнение пунктов, прикрепление результатов, просмотр прогресса |

#### 2.1.2. Действия пользователей
- Регистрация и аутентификация
- Профиль пользователя (ФИО, группа, контакты)
- Смена пароля

### 2.2. Управление группами

#### 2.2.1. Группы студентов
- Создание/редактирование/удаление групп
- Назначение преподавателя-руководителя группы
- Добавление/удаление студентов из группы
- Просмотр списка студентов в группе

#### 2.2.2. Структура группы
| Поле | Тип | Описание |
|------|-----|----------|
| ID | int | Уникальный идентификатор |
| Name | string | Название группы (напр., "ИВТ-301") |
| TeacherID | int | ID преподавателя-руководителя |
| Course | int | Курс обучения |
| Year | int | Год поступления |

### 2.3. Управление курсовыми работами

#### 2.3.1. Статусы курсовой работы
- `planning` — планирование
- `in_progress` — в процессе
- `review` — на проверке
- `completed` — завершена
- `failed` — не сдана

#### 2.3.2. Информация о курсовой
| Поле | Тип | Описание |
|------|-----|----------|
| ID | int | Уникальный идентификатор |
| StudentID | int | ID студента |
| Title | string | Тема курсовой |
| Description | text | Описание/постановка задачи |
| Status | string | Текущий статус |
| StartDate | date | Дата начала |
| Deadline | date | Общий дедлайн сдачи |
| CreatedAt | timestamp | Дата создания |
| UpdatedAt | timestamp | Дата последнего обновления |

### 2.4. План курсовой работы (Пункты)

#### 2.4.1. Структура пункта
| Поле | Тип | Описание |
|------|-----|----------|
| ID | int | Уникальный идентификатор |
| ThesisID | int | ID курсовой работы |
| Title | string | Название пункта (напр., "Анализ литературы") |
| Description | text | Детальное описание задания |
| Order | int | Порядковый номер |
| Deadline | date | Дедлайн по пункту |
| Status | string | Статус (pending/in_progress/done/rejected) |
| CreatedAt | timestamp | Дата создания |
| UpdatedAt | timestamp | Дата обновления |

#### 2.4.2. Действия с пунктами
- Создание плана по пунктам (преподаватель)
- Просмотр плана (студент)
- Изменение статуса пункта студентом
- Редактирование/удаление пунктов (преподаватель)
- Пересортировка пунктов

### 2.5. Результаты и вложения

#### 2.5.1. Результат выполнения пункта
| Поле | Тип | Описание |
|------|-----|----------|
| ID | int | Уникальный идентификатор |
| PointID | int | ID пункта курсовой |
| StudentID | int | ID студента |
| Content | text | Текстовое описание результата |
| FileURL | string | Путь к прикрепленному файлу |
| FileName | string | Имя файла |
| SubmittedAt | timestamp | Дата отправки |
| Review | text | Комментарий преподавателя |
| ReviewStatus | string | Статус проверки (pending/approved/rejected) |
| ReviewedAt | timestamp | Дата проверки |
| ReviewedBy | int | ID преподавателя |

#### 2.5.2. Функционал результатов
- Прикрепление файлов (документы, изображения, архивы)
- Текстовое описание результата
- Отправка на проверку
- Просмотр комментариев преподавателя
- Повторная отправка после доработки

### 2.6. Уведомления и дедлайны

#### 2.6.1. Типы уведомлений
- Напоминание о приближающемся дедлайне (за 3 дня, 1 день)
- Новый пункт в плане курсовой
- Результат проверен преподавателем
- Комментарий от преподавателя

#### 2.6.2. Каналы уведомлений
- Внутренние (в интерфейсе системы)
- Email (опционально)

### 2.7. Статистика и прогресс

#### 2.7.1. Для студента
- Общий прогресс курсовой (% выполнения)
- Количество пунктов: всего/выполнено/в процессе/просрочено
- Ближайшие дедлайны

#### 2.7.2. Для преподавателя
- Прогресс по каждому студенту группы
- Студенты с просроченными дедлайнами
- Ожидающие проверки результаты

## 3. Структура базы данных

### 3.1. Схема БД

```sql
-- Пользователи
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL,
    full_name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Группы
CREATE TABLE groups (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) UNIQUE NOT NULL,
    teacher_id INT REFERENCES users(id),
    course INT NOT NULL,
    year INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Студенты в группах
CREATE TABLE group_members (
    id SERIAL PRIMARY KEY,
    group_id INT REFERENCES groups(id) ON DELETE CASCADE,
    student_id INT REFERENCES users(id) ON DELETE CASCADE,
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(group_id, student_id)
);

-- Курсовые работы
CREATE TABLE theses (
    id SERIAL PRIMARY KEY,
    student_id INT REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(500) NOT NULL,
    description TEXT,
    status VARCHAR(50) DEFAULT 'planning',
    start_date DATE,
    deadline DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Пункты курсовой работы
CREATE TABLE thesis_points (
    id SERIAL PRIMARY KEY,
    thesis_id INT REFERENCES theses(id) ON DELETE CASCADE,
    title VARCHAR(500) NOT NULL,
    description TEXT,
    "order" INT NOT NULL,
    deadline DATE,
    status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Результаты выполнения пунктов
CREATE TABLE point_results (
    id SERIAL PRIMARY KEY,
    point_id INT REFERENCES thesis_points(id) ON DELETE CASCADE,
    student_id INT REFERENCES users(id),
    content TEXT,
    file_url VARCHAR(500),
    file_name VARCHAR(255),
    submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    review TEXT,
    review_status VARCHAR(50) DEFAULT 'pending',
    reviewed_at TIMESTAMP,
    reviewed_by INT REFERENCES users(id)
);

-- Уведомления
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(255) NOT NULL,
    message TEXT NOT NULL,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. API Endpoints

### 4.1. Аутентификация
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/auth/register | Регистрация |
| POST | /api/auth/login | Вход |
| POST | /api/auth/logout | Выход |
| GET | /api/auth/me | Текущий пользователь |

### 4.2. Группы
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/groups | Список групп |
| POST | /api/groups | Создание группы |
| GET | /api/groups/{id} | Информация о группе |
| PUT | /api/groups/{id} | Редактирование группы |
| DELETE | /api/groups/{id} | Удаление группы |
| POST | /api/groups/{id}/members | Добавить студента |
| DELETE | /api/groups/{id}/members/{studentId} | Удалить студента |

### 4.3. Курсовые работы
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/theses | Список курсовых (фильтр по роли) |
| POST | /api/theses | Создание курсовой |
| GET | /api/theses/{id} | Информация о курсовой |
| PUT | /api/theses/{id} | Редактирование курсовой |
| DELETE | /api/theses/{id} | Удаление курсовой |
| GET | /api/theses/{id}/points | Пункты курсовой |

### 4.4. Пункты курсовой
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/theses/{id}/points | Создать пункт |
| GET | /api/points/{id} | Информация о пункте |
| PUT | /api/points/{id} | Редактирование пункта |
| DELETE | /api/points/{id} | Удаление пункта |
| PUT | /api/points/{id}/status | Обновить статус |

### 4.5. Результаты
| Метод | Путь | Описание |
|-------|------|----------|
| POST | /api/points/{id}/results | Отправить результат |
| GET | /api/points/{id}/results | Результаты пункта |
| GET | /api/results/{id} | Информация о результате |
| PUT | /api/results/{id} | Редактировать результат |
| PUT | /api/results/{id}/review | Проверить результат |

### 4.6. Уведомления
| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/notifications | Список уведомлений |
| GET | /api/notifications/unread | Непрочитанные |
| PUT | /api/notifications/{id}/read | Отметить как прочитанное |
| PUT | /api/notifications/read-all | Отметить все как прочитанные |

## 7. Нефункциональные требования

### 7.1. Производительность
- Время отклика API < 500 мс
- Поддержка до 1000 одновременных пользователей

### 7.2. Безопасность
- Хранение паролей в хешированном виде (bcrypt)
- JWT-токены для аутентификации
- Валидация входных данных
- Защита от SQL-инъекций (prepared statements)
- Ограничение размера загружаемых файлов (макс. 10 МБ)

### 7.3. Надёжность
- Обработка ошибок и логирование
- Транзакции для критических операций
- Резервное копирование БД

### 7.4. Масштабируемость
- Модульная архитектура
- Возможность добавления новых ролей
- Поддержка множественных файлов на результат
